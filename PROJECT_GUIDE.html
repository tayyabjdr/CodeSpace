Overview

A reference for every named piece of CodeSpace — so you can point at any of it by name without guessing.

CodeSpace is a Windows-only Electron desktop app. It runs a grid of xterm.js terminals, each one backed by a real node-pty PTY hosting the Claude CLI. Multiple workspaces can run side by side in one window — switching unmounts the xterm UI but the PTYs keep buffering, so a hidden agent can finish a task and chime even while you're elsewhere.

Mental model. The whole app boils down to: Workspace → Terminals → PTY → xterm. Workspaces own persistence and PTY lifecycle; terminals are the visible panes; the PTY pool decouples those two so React mounts/unmounts don't affect the shell.

Main process src/main/

index.js

BrowserWindow factory. Notable behaviors:

  • Dev userData isolation — when ELECTRON_RENDERER_URL is set, redirects userData to %APPDATA%/CodeSpace-dev/ so a running dev session can't clobber the installed app's workspaces.json.
  • 1400×900, frameless, sandbox. Window has minWidth:720, minHeight:600.
  • Window IPC: win:minimize, win:maximize, win:close, win:ensureMaximized, win:ensureRestored, win:maximize-changed (push event).
  • uncaughtException swallow for known node-pty Windows errors: Cannot resize a pty that has already exited, Cannot write/kill ..., AttachConsole failed. Anchored to start of message so unrelated errors still surface.
  • CSP applied only in production (Vite HMR needs 'unsafe-eval' in dev). Locked-down: default-src 'self', no remote anything except data: for fonts/images.
  • will-quit hold — preventDefault, run killAllSessions(), wait 600 ms for conpty teardown, then app.exit(0). Otherwise child powershell.exe / claude.exe may orphan.
  • Clipboard capped at 1 MB writes (CLIPBOARD_WRITE_MAX_BYTES).

pty-manager.js

Single source of truth for live PTYs.

  • const sessions = new Map() — id → IPty.
  • Three shells: powershell, cmd, claude (default for new agents). The Claude shell runs powershell.exe -NoLogo -NoProfile -Command 'claude --dangerously-skip-permissions'.
  • checkClaudeAvailable() uses where claude.exe once per process; result cached.
  • Trust prompt auto-accept — for the claude shell, the first chunk containing "Yes, I trust this folder" triggers an auto-press of Enter (80 ms later).
  • Defensive write/resize/kill — every call checks proc._exited first, then wraps the native call in try/catch. onExit sets the flag.
  • killAllSessions() iterates a snapshot of keys (avoids mutation-during-iteration).

ipc-handlers.js

Registers all renderer-callable channels. Validates renderer input (isValidCwd, assertAbsPath) before crossing into privileged code. PTY data fans out via per-id channels: pty:data:<id> and pty:exit:<id>.

  • Tracks ptyDisposables: Map<ptyId, [dataDisp, exitDisp]> — disposed on pty:kill and on renderer destroyed.
  • shell:openExternal rejects anything not http(s)://, caps URL at 2048 chars.
  • If Claude isn't on PATH, pty:create with shell:'claude' resolves with { error: 'claude-missing' }; renderer surfaces a friendly card in useTerminal.js.

workspaces-store.js

Persisted state lives in app.getPath('userData')/workspaces.json.

  • Schema (per workspace): { id, name, dir, agentCount, editor? }. editor sub-object: { open, file, line, width }.
  • agentCount clamped to [1, 8] on load.
  • Atomic write — write to .tmp, then fs.rename. Power-loss-safe.
  • Corrupt-quarantine — JSON parse fail or non-array workspaces → rename original to workspaces.json.corrupt-<ISO timestamp>, return { workspaces:[], activeWorkspaceId:null }. Path surfaced via consumeCorruptBackupNotice() (one-shot).

auto-namer.js

Generates 3–5 word terminal titles by shelling out to the user's already-authenticated Claude CLI (claude -p --dangerously-skip-permissions) — piggybacks on the user's Claude Max subscription instead of needing a separate API key.

  • 30 s timeout, spawn(...) with shell:true, windowsHide:true.
  • System prompt: "You name terminal tabs. Reply with 3 to 5 words, Title Case…".
  • Sanitizes: trims, strips quotes, drops trailing .!?, caps at 40 chars.
  • hasKey() → still named that for renderer compat, now answers "is claude.exe on PATH".

auto-updater.js

electron-updater. Skipped in dev. Configured for GitHub Releases (publish.provider: 'github' in package.json).

  • autoDownload:true, autoInstallOnAppQuit:true.
  • First check delayed 10 s after launch (don't race PTY spawn).
  • On update-downloaded → win.webContents.send('update:ready', { version }) → renderer's UpdateToast shows.
  • Errors are console-only (no UI failure path by design).

editor-fs.js

File I/O for the editor pane.

  • MAX_BYTES = 20 MB — files larger return { ok:false, reason:'too-large' }.
  • Binary probe — first 8 KB scanned for 0x00; if found → reason:'binary'.
  • Failure reasons: not-found | denied | too-large | binary | unknown. Renderer copy lives in EditorPane.jsx (REASON_COPY).

App.jsx — top-level state

Holds:

workspaces: Workspace[]   // see § Workspace state shape
activeId: string | null
pendingDelete: string | null              // workspace id awaiting confirm
pendingDirtyAction: { kind, payload }     // unsaved-edit prompt action

Key behaviors layered on top:

  • Lazy spawn: terminals are created in an useEffect keyed only on activeId the first time you visit a workspace. On reload spawned:false; only {id, name, dir, agentCount} are persisted.
  • Persistence: 500 ms debounced save (PERSIST_DEBOUNCE_MS). Drafts (unconfigured:true) are filtered out.
  • Done-tracker sync: useEffect rebuilds the termId → ptyId map across all workspaces and pushes it into doneTracker.syncTracked + autoNamer.syncTracked. Hidden workspaces still chime.
  • Attended-check: doneTracker.setAttendedCheck returns true only when (workspace is active) && (pane is focused) && (document.hasFocus()). Without that last clause, alt-tabbing away suppresses sounds for the focused pane.
  • Delete workspace: persistence saved before PTY teardown. PTY kills are staggered 80 ms apart to dodge a CRT assertion node-pty throws when many conpty children exit in the same tick.
  • Dirty-prompt: switching workspace / opening a new file / closing the editor while editor.dirty triggers ConfirmDialog with Save / Discard / Cancel.

useTerminal.js

Wires xterm.js to the PTY pool. Owns:

  • Mount: spawn PTY at the renderer's measured cols × rows, register Ctrl+V/Ctrl+Shift+C clipboard interceptors, attach to pool.
  • Resize: ResizeObserver → fitAddon.fit() → resizePty.
  • Ctrl+wheel font zoom: rAF-coalesces wheel ticks into a single onFontSizeChange step, then debounces fit()+SIGWINCH by FONT_RESIZE_DEBOUNCE_MS = 180 so a scroll burst doesn't stack TUI redraws.
  • Two link providers — one for http(s):// URLs (Ctrl+click → shell:openExternal), one for parsed file paths (Ctrl+click → resolve via path system → open in editor pane).
  • data-modifierCtrl attribute toggled on the host so .xterm-link picks up cursor:pointer only when Ctrl is held.

Path system — path-parser.js + path-resolver.js

  • Parser: regex matches three pattern classes in order — Windows-absolute (C:\…), POSIX-absolute (/…), relative — and rejects ranges already claimed by an earlier pattern. Strips trailing punctuation but preserves :line / :line:col suffixes. Filters out flags, version strings, and 1.2.3-shaped tokens.
  • Resolver: pure async function. Tries the path against the focused pane's cwd, falls back to workspaceDir. Uses electronApi.pathExists for probing. Returns { path, line, col } or null.

PTY data flow decoupled lifecycles

The single most important architectural piece. Workspace switches must not kill PTYs.

Concrete sequence: workspace switch

  1. User clicks workspace B in the sidebar.
  2. App.jsx sets activeId = B.
  3. Workspace A's TerminalPane components unmount → useTerminal's cleanup runs detach() + term.dispose(). PTY is not killed.
  4. Workspace B's first-visit triggers lazy spawn (in App's useEffect keyed on activeId) — terminals get created in state, then mounted.
  5. For each new pane, useTerminal calls ptyPool.createPty if ptyId is null, else ptyPool.attach; the ring buffer is replayed into xterm so the user sees recent context immediately.

Spawn throttle

Claude CLI runs on Bun; spawning many in the same tick has been observed to panic Bun's HTTP client thread init on Windows. pty-pool.createPty enforces a 200 ms minimum gap (MIN_SPAWN_GAP_MS) between createPty calls. The slot is reserved synchronously so concurrent callers each get their own delayed start.

Done detection + auto-rename silence-based

How "done" is detected

Implemented in done-tracker.js, module-scope (not per-component) so it survives unmount.

  1. User types Enter in a pane → noteUserInput sees \r or \n → adds termId to awaiting.
  2. Each PTY data chunk (while awaiting) resets a 4 s silence timer (DONE_SILENCE_MS).
  3. When the timer fires: pane is "done" — if not attended (the active+focused pane in a focused window), play the ding, add to doneSet, notify subscribers.
  4. Either way, fire doneListeners — auto-namer subscribes here.
  5. Focusing the pane (or window.focus with that pane focused) calls noteFocus which clears doneSet for that termId.

Sidebar notification dot

App.jsx computes notifyingWorkspaceIds from doneTermIds (cyan ring next to inactive workspaces with at least one done pane).

Auto-rename pipeline

Renderer-side auto-namer.js subscribes to doneTracker.onDone:

  1. Skip if pane is pinned (manual name) or already in flight.
  2. Read PTY ring (ptyPool.getRing) → stripAnsi → take last 4096 chars.
  3. Skip if tail is identical to last attempt (no-op suppress).
  4. IPC agentName.summarize(tail) — main spawns claude -p with the system prompt.
  5. Sanitized result emitted via subscribers; App.jsx writes autoName onto the terminal.

Display priority

In TerminalPane.jsx: name?.trim() || autoName?.trim() || "Agent NN" (where NN is agentNum zero-padded).

Done sound

done-sound.js: singleton AudioContext, 440 Hz triangle wave with 60 ms attack and 950 ms exponential release. Volume routed through volume-store.js (localStorage, codespace.volume.v1, 0–100 → gain 0–0.4 linear).

Editor pane flow CodeMirror 6, lazy-loaded

The editor pane is React.lazy-imported (CodeMirror is heavy and not needed at first paint).

Opening a file

  1. User Ctrl+clicks a path in xterm → useTerminal's path linkProvider activates.
  2. resolvePath (renderer) checks focused cwd, then workspaceDir, via editor.pathExists.
  3. onOpenFile({path, line, col}) bubbles up to App's openFileInEditor.
  4. If editor is dirty and current file ≠ new file → set pendingDirtyAction = { kind: 'open-file', payload } → ConfirmDialog with Save/Discard/Cancel.
  5. Otherwise setEditorPatch({ open:true, file:path, line, dirty:false }).
  6. App's useEffect on editorFile calls editor.readFile → either renders content or an error card (REASON_COPY in EditorPane.jsx).

useEditor hook

  • Plain mode threshold: 2 MB (PLAIN_MODE_THRESHOLD) — disables line numbers, active-line highlight, syntax highlighting.
  • Languages auto-detected by extension: js/jsx/mjs/cjs/ts/tsx → JS w/ JSX + TypeScript flag; json, md/markdown, html/htm, css.
  • Font size lives in a Compartment so tweaking it doesn't rebuild the EditorView.
  • Ctrl+S binding → onSave(view.state.doc.toString()).
  • Dirty tracking: compares current doc against lastSavedRef; emits onDirtyChange.
  • jumpToLine(line) exposed for initialLine on file open.

Resizer

EditorResizer.jsx — pointer-captured drag, double-click resets to default. Width clamped via editor-state.clampWidth against bodyWidth: [320 px, body × 0.7], default body × 0.45.

External-file chip

If editor file path doesn't start with workspaceDir (case insensitive) the header shows an EXTERNAL chip. Plain-mode files show a PLAIN chip.

Workspace state shape canonical reference

Only the persisted fields survive across launches. Everything else is session-only — repopulated lazily.

type Workspace = {
  // Persisted (workspaces.json)
  id: string                 // crypto.randomUUID()
  name: string               // user-edited; defaults to folder basename
  dir: string                // absolute path; cwd for spawned PTYs
  agentCount: number         // 1..8 (clamped on load)
  editor?: {                 // optional — only if user opened the editor
    open: boolean
    file: string | null      // last-open file
    line: number | null      // jump-to line
    width: number            // px, clamped to [320, body*0.7]
  }

  // Session-only (recreated each launch)
  terminals: Terminal[]      // [] until lazy spawn
  agentCounter: number       // monotonic for naming "Agent NN"
  focusedTerminalId: string | null
  spawned: boolean           // becomes true on first activation
  fontSize: number           // 8..28, default 13
  unconfigured?: boolean     // true for *drafts* — see § Onboarding
}

type Terminal = {
  id: string                 // crypto.randomUUID() — also the autoNamer key
  shell: 'claude' | 'powershell' | 'cmd'
  agentNum: number           // display index (1-based)
  cwd: string
  ptyId: string | null       // populated by handlePtyReady
  autoName: string | null    // last summarize() result
  name?: string              // manual rename — pins the pane (no auto-rename)
}

Pinning rule: terminal.name with a non-empty trimmed value → pane is "pinned" → auto-namer skips it (auto-namer.js → isPinned).

Persistence

File location

  • Production: %APPDATA%\CodeSpace\workspaces.json
  • Dev: %APPDATA%\CodeSpace-dev\workspaces.json (redirected in main/index.js)

Write path

  1. Renderer dispatches state-shape changes (workspace add/delete, name edit, agent count, editor open/file/line/width).
  2. App.jsx debounces by 500 ms (PERSIST_DEBOUNCE_MS) → invokes workspaces:save.
  3. workspaces-store.saveWorkspaces writes to workspaces.json.tmp, then fs.rename → atomic.
  4. Workspace delete bypasses the debounce — saves before killing PTYs.

Read path

  1. App mount → loadWorkspaces().
  2. If ENOENT → empty state.
  3. If JSON parse error or invalid shape → quarantine (rename to .corrupt-<ISO>) → empty state. Path is exposed once via consumeCorruptBackupNotice and bundled into the workspaces:load response (currently consumed only as a side-effect; no UI banner yet).
  4. Otherwise sanitize + clamp: id/name/dir → String, agentCount → max(1, min(8, n)), editor rebuilt from {open, file, line, width}.

Volume persistence

Separate from workspaces. localStorage["codespace.volume.v1"] holds { volume: 0–100 }.

Onboarding + drafts

Two modes for the same component

Onboarding.jsx renders in two modes:

  • mode="standalone" — full-screen first-launch: shown when workspaces.length === 0. Submit → onLaunch(count, dir, name) → App creates the workspace and replaces this with the main shell.
  • mode="embedded" — rendered inside the workspace grid area when the active workspace is a draft (unconfigured:true). Submit → onLaunch → handleInitializeDraft mutates the existing draft into a real workspace in one state update.

Draft lifecycle

  1. Sidebar "+" or "New Workspace" → handleStartDraft creates { id, name:'Untitled', dir:'', agentCount:0, unconfigured:true, ... }.
  2. Drafts do not persist (filtered out by App.jsx's save effect).
  3. Live name typing bubbles up via onNameChange → sidebar label updates as you type.
  4. Initialize → name + dir + count populated, unconfigured:false, terminals spawned in same setState.
  5. Discard / close button → handleDiscardDraft removes it.

Folder default

Project directory defaults to the OS Desktop path (app:getDesktopPath). Workspace name auto-fills from the selected folder's basename — but only if the user hasn't typed their own.

Drag-to-swap

  • Header is the drag handle (draggable={!editing}).
  • Pane is the drop target.
  • MIME type: application/x-codespace-terminal carries the source termId.
  • Reorder is a renderer-only swap of the active workspace's terminals array — same key={t.id} keeps xterm in place.

OS file drop

Same drop target also accepts OS files (Files dataTransfer type). Dropped paths are quoted (if they contain space or quote) and written into the PTY so Claude CLI can attach them.

Aesthetic rules

  • Animations are one-shot 200–500 ms ease curves. Avoid infinite/looping animations — they read as flicker on this palette.

Icon pipeline

SVG source at build/icon.svg → scripts/build-icon.js rasterizes at 16/24/32/48/64/128/256 with sharp (density: 384 for stroke detail) → png-to-ico bundles into build/icon.ico. Used by both electron-builder (production .exe) and the dev BrowserWindow (taskbar).

node-pty rebuild

node-pty is a native module. Postinstall runs electron-rebuild -f -w node-pty. On a fresh checkout this requires Python 3.12 + setuptools and a patched node_modules/node-pty/deps/winpty/src/winpty.gyp (see project memory).

Auto-update

  • Provider: GitHub Releases (owner tayyabjdr, repo CodeSpace).
  • Skipped in dev (ELECTRON_RENDERER_URL set).
  • First check 10 s after launch; downloads in background.
  • On update-downloaded: UpdateToast shows the new version + "Restart now" / "Later".
  • "Restart now" → autoUpdater.quitAndInstall(). "Later" hides until next launch — at next launch the update installs on quit (autoInstallOnAppQuit:true).
  • Errors are silent (console-only) by design.

Tests

vitest with jsdom. Coverage exists for path parsing/resolving, IPC handlers, editor FS, editor state, PTY manager, and several React components. Use npm run test:watch while editing.

Windows quirks

Environment realities, not bugs to fix.

  • conpty_console_list_agent.js: Error: AttachConsole failed — node-pty child worker fails to attach a console. Swallowed. Cosmetic.
  • Multiple PTYs killed in quick succession can seg-fault main (exit 139). Mitigated by save-before-kill (handleConfirmDelete) and the staggered 80 ms kill loop.
  • Sometimes Claude spawns visibly outside the app — same conpty fallback. Tell the user to close the orphan; the in-app pane is fine.
  • node-pty native rebuild on fresh checkout needs Python 3.12 + setuptools and a patched winpty.gyp.
  • Switching from powershell.exe -Command 'claude ...' to cmd.exe /c claude may avoid some fallback paths but is untested.
